# Upgrade guide: v0.1.0 ‚Üí v0.1.1

**v0.1.1** adds **`agent-pip`**: persistent Python package installs on the `/home/agent` volume, wired into the Hermes runtime (same idea as `npm install -g` with `NPM_CONFIG_PREFIX=~/.local`).

There is **no volume path change** from v0.1.0. You only need this guide if you upgraded from v0.1.0 (or installed Python libs into the ephemeral image venv).

## What changed

| Topic | v0.1.0 | v0.1.1 |
|-------|--------|--------|
| Extra Python libs | Docs suggested `pip install --user` (often fails: venv blocks `--user`, Debian PEP 668 blocks system pip) | **`agent-pip install <pkg>`** ‚Üí `~/.local/lib/pythonX.Y/site-packages` on the volume |
| Hermes imports user libs | Not wired | Entrypoint writes **`hermes-user-local.pth`** in the Hermes venv pointing at the volume path |
| CLI | ‚Äî | **`/usr/local/bin/agent-pip`** (wrapper around Hermes venv `pip --target`) |

Volume layout under `/home/agent` is unchanged: `.hermes/`, `.local/`, `workspace/`, etc.

## Pre-flight

- [ ] Note current image tag (`v0.1.0`) and `HERMES_REF` build arg
- [ ] List Python packages you added manually (especially anything installed with `/opt/hermes-agent/venv/bin/pip install` ‚Äî **not persisted** across image rebuilds)
- [ ] Optional: back up the Railway volume (snapshot or tar) before redeploy

## Step 1 ‚Äî Deploy v0.1.1

1. Bump / pull image **`ghcr.io/<owner>/hermes-agent-docker:v0.1.1`** (or rebuild locally with `VERSION=0.1.1`).
2. Keep the volume mount at **`/home/agent`** (same as v0.1.0).
3. Leave Railway **start command** empty (use image `ENTRYPOINT` + `CMD`).
4. Restart the service.

On first boot, the entrypoint logs a line like:

```text
[agent-pip] volume site-packages: /home/agent/.local/lib/python3.12/site-packages
```

(Exact `3.12` matches the Hermes venv Python version.)

## Step 2 ‚Äî Reinstall ephemeral Python packages

SSH in as **`agent`** (`su - agent` from a root Railway shell).

### Inventory (optional)

Packages only in the image venv (lost on rebuild until reinstalled):

```bash
/opt/hermes-agent/venv/bin/pip list | grep -iE 'hindsight|your-pkg'
ls -la ~/.local/lib/python*/site-packages 2>/dev/null | head
```

### Install with agent-pip (persistent)

```bash
agent-pip install "hindsight-client>=0.4.22"
# agent-pip install <other-package>
```

Verify:

```bash
TARGET=$(/opt/hermes-agent/venv/bin/python -c "import sys; print(f'/home/agent/.local/lib/python{sys.version_info.major}.{sys.version_info.minor}/site-packages')")
/opt/hermes-agent/venv/bin/python -c "import hindsight_client; print('ok')"
agent-pip list
ls "$TARGET" | head
```

Update **`~/.hermes/migration-reinstall.txt`** (if you use it) ‚Äî replace venv `pip install` lines with:

```text
agent-pip install hindsight-client>=0.4.22
```

## Step 3 ‚Äî Migrate mistaken pip trees (if any)

If an older session left **`site-packages`** or **`lib/python*`** under **`~/.hermes/`** (pre‚Äìv0.1.0 workaround), merge into **`~/.local/`** and remove duplicates. Do **not** move Hermes state (`state.db`, `config.yaml`, `skills/`, ‚Ä¶).

Example (review before running):

```bash
# Only if you have ~/.hermes/lib or ~/.hermes/site-packages from old hacks:
# rsync -a ~/.hermes/lib/ ~/.local/lib/ 2>/dev/null || true
```

Then reinstall anything missing with **`agent-pip install`**.

## Step 4 ‚Äî Validation

```bash
export HOME=/home/agent HERMES_HOME=/home/agent/.hermes
hermes doctor
agent-pip list
test -f "$(/opt/hermes-agent/venv/bin/python -c 'import site; print(site.getsitepackages()[0])')/hermes-user-local.pth" && echo "pth ok"
```

Restart the container once and confirm imports still work (`.pth` is recreated every start).

## What not to use

| Command | Problem |
|---------|---------|
| `pip install <pkg>` (default `pip` in venv) | Installs into **image** venv ‚Äî ephemeral |
| `pip install --user <pkg>` (venv pip) | **Disabled** in virtualenv |
| `/usr/bin/python3 -m pip install --user` | **PEP 668** externally-managed on Debian |
| `pip install` into `~/.hermes/` | Wrong layer ‚Äî use **`agent-pip`** or a documented skill venv under `~/.hermes/venvs/` |

## npm / other runtimes (unchanged)

- **Node globals:** `npm install -g <pkg>` ‚Üí `~/.local` via `NPM_CONFIG_PREFIX`
- **Bun:** `~/.bun`
- **Baked deps:** Dockerfile `uv pip install` into `/opt/hermes-agent/venv` (e.g. `faster-whisper`) ‚Äî image-only, not operator installs
